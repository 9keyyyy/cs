### 컴퓨터가 이해하는 정보

- 컴퓨터는 자바, C++, 파이썬 같은 프로그래밍 언어를 직접 이해하지 못함 → 이해할 수 있는 것은 데이터와 명령어
- 데이터는 숫자, 문자, 이미지, 동영상과 같은 정적인 정보. 명령어는 이 데이터를 활용하는 정보. 데이터는 명령어에 종속적인 정보이며, 명령어의 대상이자, 명령어의 재료라고 할 수 있음
- 컴퓨터는 기본적으로 0과 1만 이해할 수 있음 (곧 데이터와 명령어도 0과 1)
- 명령어를 이해하고 실행하는 주체는 CPU → CPU 종류에 따라 실행 가능한 세부적인 명령어의 종류와 처리의 양상이 달라질 수 있음

### 컴퓨터의 핵심 부품

CPU, 메모리, 캐시 메모리, 보조기억장치, 입출력장치

- **CPU**
    - 데이터와 명령어를 읽어 들이고, 해석하고, 실행하는 부품
        - 산술논리연산장치 (ALU, Arithmetic and Logic Unit): 연산을 수행하는 일종의 계산기
        - 제어장치 (CU, Control Unit): 명령어를 해석해서 제어 신호를 보내는 장치
        - 레지스터 (Register): CPU 내부의 작은 임시 저장장치. 여러 레지스터가 존재하며 각기 다른 이름과 역할 가짐
- **메모리와 캐시 메모리**
    - 메인 메모리: RAM, ROM. 일반적으론 RAM을 메인 메모리라고 지칭.
        - 메모리는 CPU가 읽어들이는 정보를 저장하는 장치. 실행 중인 프로그램을 구상하는 데이터와 명령어를 저장하는 부품
        - 메인 메모리(RAM)는 휘발성 저장장치로, 전원이 꺼지면 데이터가 소멸됨
    - 캐시 메모리:
        - CPU와 메인 메모리 사이에 위치하는 저장장치
        - CPU가 메모리에 저장된 값에 더 빠르게 접근하기 위한 목적
        - CPU 내부에 위치하기도 하고 외부에 위치하기도 하는 등 여러 종류가 존재
- **보조 기억 장치**
    - 전원이 꺼져도 데이터가 유지되는 비휘발성 저장장치
    - 대표적인 예: CD, DVD, 하드디스크, 플래시메모리
    - 현대 컴퓨터에서는 주로 하드디스크와 SSD 사용
    - 메모리는 실행 중인 프로그램 저장, 보조기억장치는 보관할 프로그램 저장
    - CPU는 보조기억장치의 프로그램을 직접 실행할 수 없고, 메모리로 올린 후 실행 가능
- **입출력장치**
    - 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환하는 장치
    - 컴퓨터에 정보를 입력하는 장치를 입력장치라고 함
- **컴퓨터 핵심 부품과 메인보드**
    - 컴퓨터 핵심 부품(CPU, 메모리 등)은 메인보드(마더보드)에 연결되어 있음
    - 메인보드에는 다양한 슬롯과 연결 단자가 있으며, 부품 간 정보를 주고받는 통로를 '버스'라고 함
    - 시스템 버스는 가장 중요한 버스로, 사람의 척추와 같은 역할을 함
- **저장장치의 계층구조**
    - CPU에 가까운 순서: CPU 내부 레지스터 → 캐시 메모리 → 주기억장치(메모리) → 보조기억장치
    - CPU에 가까울수록 속도 빠르고, 비용 높으며, 용량이 작음 (피라미드 구조)
    - 캐시 메모리는 L1, L2, L3로 구분됨
