## 면접 예상 질문

### Q. ARP란 무엇이고 왜 필요한가요?

ARP(Address Resolution Protocol)는 **IP 주소를 MAC 주소로 변환**하는 프로토콜입니다.

네트워크 계층(L3)은 IP 주소로 통신하지만, 실제 데이터를 전송하는 데이터링크 계층(L2)은 **MAC 주소**를 사용합니다. 따라서 같은 네트워크 내에서 프레임을 보내려면 상대방의 MAC 주소를 알아야 하며, 이때 ARP가 사용됩니다.

**ARP 동작 과정**:
1. **ARP Request**: 송신자가 "IP 192.168.1.10의 MAC 주소를 알려주세요"라는 **브로드캐스트** 메시지를 LAN 전체에 전송
2. **ARP Reply**: 해당 IP를 가진 호스트만 자신의 MAC 주소를 **유니캐스트**로 응답
3. **ARP 캐시 저장**: 응답받은 IP-MAC 매핑을 ARP 테이블에 캐싱 (일정 시간 후 만료)

> 다른 서브넷으로 보낼 때는 목적지 MAC을 **기본 게이트웨이(라우터)의 MAC**으로 설정합니다. IP는 최종 목적지이지만, MAC은 다음 홉(next hop)의 주소입니다.

### Q. ARP 스푸핑이란?

ARP 스푸핑(Spoofing)은 **위조된 ARP Reply를 보내 다른 호스트의 ARP 테이블을 조작하는 공격**입니다.

**공격 원리**:
1. 공격자가 게이트웨이의 IP에 대해 "내 MAC이 게이트웨이의 MAC이다"라는 위조 ARP Reply를 피해자에게 전송
2. 피해자의 ARP 테이블이 오염되어 게이트웨이로 보내는 트래픽이 공격자에게 전달됨
3. 공격자는 이 트래픽을 가로채(스니핑)거나 변조한 뒤 실제 게이트웨이로 포워딩 → **중간자 공격(MITM)**

**방어 방법**:
- **정적 ARP 엔트리**: 중요 장비의 ARP를 수동으로 고정
- **Dynamic ARP Inspection (DAI)**: 스위치에서 ARP 패킷의 정당성을 검증
- **HTTPS/TLS**: 트래픽이 가로채여도 암호화로 내용 보호

### Q. 서브넷 마스크의 역할은?

서브넷 마스크는 IP 주소에서 **네트워크 부분과 호스트 부분을 구분**하는 32비트 값입니다.

```
IP:          192.168.1.130  = 11000000.10101000.00000001.10000010
서브넷 마스크: 255.255.255.0  = 11111111.11111111.11111111.00000000
                                ├── 네트워크 부분 ──┤├ 호스트 ┤
```

- **1인 비트**: 네트워크 주소 부분 → 같은 네트워크인지 판별에 사용
- **0인 비트**: 호스트 주소 부분 → 같은 네트워크 내에서 개별 장치 식별

**CIDR 표기법**: `/24`는 상위 24비트가 네트워크 부분임을 의미 (= 서브넷 마스크 255.255.255.0)

서브넷 마스크를 통해 하나의 네트워크를 여러 서브넷으로 분할하면:
- **브로드캐스트 도메인 축소** → 네트워크 효율 향상
- **IP 주소 낭비 감소** → 필요한 만큼만 할당
- **보안 강화** → 서브넷 간 접근 제어 가능

### Q. 192.168.1.130/25의 네트워크 주소와 브로드캐스트 주소는?

`/25`이므로 상위 25비트가 네트워크, 하위 7비트가 호스트입니다.

```
IP:     192.168.1.10000010  (130)
마스크:  192.168.1.1|0000000  (/25 → 상위 25비트)
                    ↑ 여기서 분리
```

- **서브넷 마스크**: 255.255.255.128 (`11111111.11111111.11111111.10000000`)
- **네트워크 주소**: IP AND 마스크 = `192.168.1.128` (호스트 비트 모두 0)
- **브로드캐스트 주소**: 호스트 비트 모두 1 = `192.168.1.255`
- **사용 가능 호스트 범위**: 192.168.1.129 ~ 192.168.1.254 (126개)

> `/25`는 256개의 IP를 128개씩 2개의 서브넷(0~127, 128~255)으로 나눕니다. 130은 두 번째 서브넷에 속합니다.

### Q. 같은 서브넷에 있는지 어떻게 판별하나요?

두 IP 주소에 각각 **서브넷 마스크를 AND 연산**하여 결과(네트워크 주소)가 같으면 같은 서브넷입니다.

```
예: 192.168.1.10과 192.168.1.200은 /25에서 같은 서브넷인가?

192.168.1.10   AND 255.255.255.128 = 192.168.1.0    (첫 번째 서브넷)
192.168.1.200  AND 255.255.255.128 = 192.168.1.128  (두 번째 서브넷)
→ 네트워크 주소가 다르므로 다른 서브넷
```

**같은 서브넷이면** 목적지 IP로 ARP 요청을 보내 MAC 주소를 조회한 뒤 직접 통신합니다. **다른 서브넷이면** 기본 게이트웨이(라우터)의 MAC 주소를 ARP로 조회하여 프레임을 라우터에게 전달하고, 라우터가 L3 라우팅으로 목적지 서브넷까지 전달합니다. 즉, **ARP는 어느 경우든 사용되며**, 차이는 ARP 요청의 대상이 목적지 호스트인지 게이트웨이인지입니다.
